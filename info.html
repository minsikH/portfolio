1. 메인 (입양)

  • 2) 공공데이터 API : -query 상태가 변경될 때마다 API 요청을 보내어 해당 필터에 맞는 강아지 데이터를 가져온 후, 가져온 데이터는 petList에 저장되며, 이를 Content 컴포넌트에 props로 전달하여 화면에 출력합니다.
  • 3) 페이지네이션 : -Content 컴포넌트에서 petList를 받아와서 페이지네이션 기능을 구현합니다.

1-1. 입양 -> 디테일 페이지

  • 1) 공공데이터 API : -useParams 훅을 사용하여 URL에서 동물의 고유 ID(id)를 가져오고, 이를 통해 Redux에서 해당 동물의 상세 정보를 가져오는 dispatch(petAction.getPetDetail(id))를 호출합니다.

3. 보호소

  • 1) 카카오맵API : -선택된 보호소에 따라 해당 보호소의 동물 수를 계산하고, ShelterCard 컴포넌트를 통해 보호소 정보와 관련된 동물 수를 표시합니다.
  • 2) 공공데이터 API : -useDispatch와 useSelector를 사용하여 Redux 상태 관리를 설정하고, petAction.getAllShelters()를 통해 모든 보호소 정보를 가져옵니다.

3-1. 보호소 -> 디테일 페이지

  • 1) 공공데이터 API : -petAction.getAllShelters()를 호출하여 Redux의 상태를 업데이트하고, 모든 보호소 정보를 가져옵니다.
  • 2) 카카오맵API : -Kakao 컴포넌트를 사용하여 Kakao 지도 API를 통해 선택된 보호소의 위치를 표시합니다.
  • 3) 공공데이터 API : -선택된 보호소의 careAddr을 기반으로 allPetsList에서 해당 보호소의 동물 목록을 필터링하여 가져옵니다.

4. 로그인

  • 1) 파이어베이스 : -useLogin: useLogin 훅을 사용하여 로그인 기능을 가져옵니다. 이 훅은 login 함수를 제공하여 파이어베이스를 통해 사용자가 입력한 이메일과 비밀번호로 로그인을 처리합니다.
  • 2) 조건부 렌더링 : -user의 로그인 상태를 통해 조건부로 로그인 버튼과 로그아웃 버튼을 렌더링합니다.

5. 회원가입
